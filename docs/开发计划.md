# 无尽跑酷游戏 — Codex 全自动化开发计划

## 0. 前置约定

### 0.1 开发环境
| 项 | 值 |
|----|-----|
| 引擎 | Unity 4.6.8 |
| 语言 | C#（.NET 3.5 等效，Mono） |
| 项目路径 | `D:\unitypk` |
| 文档路径 | `D:\unitypk\docs` |
| 换行符 | CRLF |
| 编码 | UTF-8 without BOM |

### 0.2 强制约束（每步必遵守）
- **无 async/await** → 用协程 `StartCoroutine()`
- **无 `?.`** → 显式 `if (obj != null)`
- **无 string interpolation** → `string.Format()` 或 `+`
- **场景加载** → `Application.LoadLevel()`（非 `SceneManager`）
- **光标控制** → `Screen.lockCursor` / `Screen.showCursor`
- **Shader** → `Self-Illumin/Diffuse`（非 `Self-Illuminated`）
- **发射颜色** → `SetColor("_Illum", ...)`（非 `_Emission`）
- **地面碰撞** → 有厚度 Cube（非 Plane），角色 Y 初始 ≥ 1.5
- **材质** → 不使用任何 `.mat` 文件，颜色全部 `renderer.material.color` 代码设置
- **粒子/音频** → 禁止使用
- **模型/贴图** → 禁止导入，仅用 `GameObject.CreatePrimitive`

需求分析文档已完成并通过多轮评审，详见 [需求分析.md](D:\unitypk\docs\需求分析.md)。

## 2. 阶段详细任务

### P0 — 项目初始化
- [ ] 在 Unity 4.6.8 中创建新项目，路径 `D:\unitypk`
- [ ] 设置 `Edit → Project Settings → Editor`：Asset Serialization = Force Text，Version Control = Visible Meta Files
- [ ] 创建目录结构：
  ```
  Assets/
  ├── Scripts/
  │   ├── Player/
  │   ├── Gameplay/
  │   ├── Skills/
  │   ├── Environment/
  │   └── UI/
  ├── Scenes/
  ├── Prefabs/
  └── Resources/
  ```
- [ ] 创建 `Assets/Editor/` 目录（后续放构建脚本）

#### 提交策略

| 时机 | 操作 |
|------|------|
| 每个阶段完成后 | `git add -A && git commit -m "P{n}: 阶段描述"` |
| 阶段内每完成一个脚本 | 建议单独 commit，便于回退 |
| 打包前 | 打 tag：`git tag -a v1.0 -m "首次可玩版本"` |

### P1 — 场景搭建 + 角色奔跑
- [ ] 创建 `Assets/Scenes/Game.unity`
- [ ] 搭建基础场景：
  - Directional Light（默认）
  - Main Camera（位置：后方俯视）
  - 三条跑道：3 个拉伸 Cube（间距 X=2，Z 方向延伸 200+），颜色左灰/中深灰/右灰
  - 地面两侧边界 Cube
- [ ] 创建角色预制体 `Assets/Prefabs/Player.prefab`：
  - Capsule（身体）+ Sphere（头部）
  - 挂载 CharacterController
  - 初始位置 Y ≥ 1.5
- [ ] 创建 `PlayerController.cs`，实现：
  - F01 自动奔跑：每帧 `transform.Translate(Vector3.forward * speed * Time.deltaTime)`
  - 速度通过 `public float speed` 暴露

### P2 — 三轨道 + 跳跃/滑铲
- [ ] 扩展 `PlayerController.cs`，新增：
  - F04 变道：三轨道 X ∈ {-2, 0, 2}，按键 A/D 平滑 Lerp 切换
  - F02 跳跃：Space/W/↑，`CharacterController.Move` 施加 Y 轴速度，接地时重置
  - F03 滑铲：S/↓/Ctrl，缩小 Capsule 高度 + 碰撞体 center 下移，松开恢复
- [ ] 角色始终面向前方，变道仅改 X

### P3 — 静态障碍物 + 碰撞即死
- [ ] 创建障碍物预制体（纯几何体组合）：
  - O01 石块：1×1×1 Cube，深灰
  - O02 墙体：3×3×0.5 Cube，暗红，横跨 2~3 轨道
  - O03 尖刺：3 个倒置小 Cylinder，红色
  - O04 断台：轨道缺口（移除该段地面碰撞体）
- [ ] 创建 `ObstacleSpawner.cs`：
  - 基于对象池，从前方 Z+50~100 范围随机生成
  - 随机选择类型和轨道位置
  - 保证至少 1 条轨道可通行
  - 障碍物超过角色后方 Z-20 回收
- [ ] 创建 `GameManager.cs`，实现：
  - F06 分数系统：距离 × 速度系数
  - 碰撞检测 → 即死（F12 死亡回溯暂留桩）
  - 状态机：Playing → Dead → Over

### P4 — 动态障碍 + 陷阱
- [ ] 创建 O05 旋转机关预制体：长条 Cube 绕中心旋转，暗黄
- [ ] 创建 `DynamicObstacle.cs`：控制旋转机关转速，与全局速度同步
- [ ] 创建 O08 掉落深坑：大段缺口，检测角色 Y < -5 → 即死
- [ ] 创建 O09 即死区域：红色 Cube 阵列，2s 开/2s 关间歇激活
- [ ] 创建 `TrapController.cs`：管理深坑和即死区域的触发逻辑
- [ ] 扩展 `ObstacleSpawner.cs`：难度阶段解锁（O01~O04 全程，O05 500m 后，O08/O09 1000m 后）

### P5 — 追击列车 + 地形塌陷
- [ ] 创建列车预制体：大型拉伸 Cube 组合（车头+车身），暗红
- [ ] 创建 `TrainController.cs`：
  - 列车初始在玩家后方 100m
  - 距离受速度影响，逼近时画面边缘红光渐强
  - 距离 < 10m → 触发即死
- [ ] 创建 `TerrainCollapse.cs`：
  - 随机选择前方轨道方块，0.5s 黄色闪烁预警后下沉消失
  - 3~5s 后恢复升起
  - 塌陷频率随难度提升

### P6 — 金币 + 双倍分数道具
- [ ] 创建金币预制体：黄色 Sphere + 持续旋转
- [ ] 创建 `CoinSpawner.cs`：地面和浮空（Y=2~3）两种金币，随机分布
- [ ] 金币碰撞检测 → 计入金币数 + 分数
- [ ] 创建 I01 双倍分数道具预制体：黄色 Cube 旋转 + 顶面 "×2"
- [ ] 创建 `PowerUpManager.cs`：拾取后 10 秒分数翻倍，HUD 进度条倒计时
- [ ] 地面和浮空两种生成方式

### P7 — 技能系统
- [ ] 创建 `Skills/SkillManager.cs`：管理冷却计时、全局互斥
- [ ] 创建 `Skills/UltimateDash.cs`：
  - S02 虚空冲刺：E 键触发，冷却 45s
  - 向前冲刺 30m，路径清障 + 无敌
- [ ] 创建 `Skills/PhantomClone.cs`：
  - S03 幻影分身：R 键触发，冷却 25s
  - 三轨道各生成半透明分身，持续 6s 拾取金币
- [ ] 创建 `Skills/TimeRewind.cs`：
  - S01 时间回溯：每 0.1s 记录位置快照（最近 3s）
  - 1500 分解锁，死亡时弹出回溯按钮
  - 点击回退至碰撞前 2s，每局限 1 次

### P8 — HUD + UI 全链路
- [ ] 创建 `Assets/Scenes/MainMenu.unity`：Canvas + Text 标题 + "开始游戏"按钮
- [ ] 创建 `UIManager.cs`：管理场景切换、面板显隐
- [ ] 创建 `HUDController.cs`：
  - 列车距离指示条（左上角）
  - 当前分数 + 金币数
  - 技能栏（底部居中）：S01 回溯图标 + E/R 冷却倒计时
  - 道具栏（顶部居中）：双倍分数进度条
- [ ] 结算界面：半透明遮罩 + 分数 + 最高分 + 重新开始/返回主菜单
- [ ] 死亡回溯弹窗（≥1500 分时显示）
- [ ] F20/F21 最高分和金币数 PlayerPrefs 持久化

### P9 — 自适应难度 + 背景滚动
- [ ] 创建 `AdaptiveDifficulty.cs`：
  - 跟踪最近 5 次存活距离
  - 平均 > 2000m → 难度 +1（上限 10）
  - 平均 < 500m → 难度 -1（下限 1）
  - 难度影响：速度、障碍密度、塌陷频率、道具生成率
- [ ] 创建 `BackgroundScroller.cs`：3~4 层 Cube 阵列视差滚动，灰/白/浅蓝
- [ ] 轨迹特效：角色奔跑时身后生成短暂 Cube 轨迹

### P10 — 调参 + 打包
- [ ] 各参数外露为 `public` 变量，便于 Inspector 调参
- [ ] 测试 4 个难度等级下的障碍组合和帧率
- [ ] 角色手感调优：跳跃高度、滑铲时长、变道速度
- [ ] 技能冷却和强度平衡
- [ ] 创建 `Assets/Editor/Builder.cs`：batchmode 构建脚本
- [ ] Windows Standalone 打包输出

## 4. 执行顺序依赖

```
P0 ──→ P1 ──→ P2 ──┬──→ P3 ──→ P4
                    │         │
                    │         ├──→ P5 ──→ P9
                    │         │
                    │         └──→ P6
                    │
                    ├──→ P7 ──────────→ P8 ──→ P10
                    │                          
                    └──→ (P6) ────────────────→ P9
```

P2 之后 P3/P6/P7 可并行。P8 需等 P6（分数数据）和 P7（技能冷却数据）。P9 需等 P5（列车距离）和 P7（技能使用频率）。P10 串行收尾。
