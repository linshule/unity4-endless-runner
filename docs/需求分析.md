# 跑酷游戏需求分析文档

---

## 1. 项目概述

### 1.1 项目名称
**Unity Endless Parkour（无尽跑酷）**

### 1.2 项目背景
本项目为 Unity 游戏开发课程实践作业，旨在通过完成一款完整的 3D 跑酷游戏，掌握 Unity 引擎的核心开发技能，包括场景搭建、物理系统、动画控制、UI 系统、游戏状态管理等。

### 1.3 游戏定位
一款快节奏的 **无尽 3D 跑酷游戏**。玩家操控角色在几何风格赛道上持续奔跑，需应对静态障碍、动态障碍和陷阱机制。角色碰撞障碍物即死。死亡时可手动触发时间回溯（1500 分解锁），回退至碰撞前，每局限 1 次。**所有视觉元素仅使用 Unity 内置基础几何体。**

### 1.4 目标平台
| 平台 | 说明 |
|------|------|
| Windows Standalone | 主要目标平台 |
| 分辨率 | 1920×1080（支持 16:9 自适应） |

### 1.5 技术栈
- **引擎**: Unity 4.6.8
- **语言**: C#（.NET 3.5 等效，Mono 运行时）
- **物理**: Unity 内置 PhysX
- **动画**: Legacy Animation System / Mecanim（视模型而定）

## 2. 功能需求

### 2.1 核心玩法（P0 — 必须实现）

| 编号 | 功能 | 描述 |
|------|------|------|
| F01 | 自动奔跑 | 角色自动向前匀速移动，速度随游戏时间递增 |
| F02 | 跳跃 | 玩家按下跳跃键（Space），角色执行跳跃动作，跨越地面障碍 |
| F03 | 滑铲 | 玩家按下下滑键（↓/S），角色执行滑铲动作，通过低位障碍 |
| F04 | 左右变道 | 玩家按下 ←→/AD，角色在三条轨道间切换 |
| F05 | 障碍物生成 | 赛道前方随机生成三类障碍物（静态/动态/陷阱），详见 §2.6 障碍物体系详表 |
| F06 | 分数系统 | 根据奔跑距离实时计分，支持道具倍率加成 |
| F07 | 速度递增 | 游戏速度随时间逐渐加快，分四阶段难度递进 |

### 2.2 游戏流程（P0）

| 编号 | 功能 | 描述 |
|------|------|------|
| F08 | 开始界面 | 显示游戏标题 + "开始游戏"按钮 |
| F09 | 游戏结束 | 碰撞障碍物后弹出结算界面，显示本次分数和最高分 |
| F10 | 重新开始 | 结算界面提供"重新开始"按钮，无需回到主菜单 |
| F11 | 暂停功能 | 按 Esc 暂停/恢复游戏 |
| F12 | 死亡回溯 | 碰撞死亡时若已解锁时间回溯（>= 1500 分），屏幕弹出回溯按钮，点击后回退至碰撞前 2 秒，黑白倒放。每局限 1 次，被列车追上不可用 |

### 2.3 创新机制（P0 — 核心差异化）

| 编号 | 功能 | 描述 |
|------|------|------|
| F13 | 地形塌陷 | 角色脚下路面随机塌陷——前方轨道方块下沉消失，持续 3~5 秒后恢复。塌陷前 0.5s 地面闪烁预警。需跳跃或变道规避 |
| F14 | 自适应难度 | 追击列车距离 + 障碍物密度 + 塌陷频率根据玩家生存表现动态调整：长时间不死则难度提升，频繁死亡则难度回降 |

### 2.4 创新技能系统（P0 — 核心差异化）

| 编号 | 技能 | 按键 | 效果 | 冷却 |
|------|------|------|------|------|
| S01 | 时间回溯 | 死亡时点击触发 | 达到 1500 分后解锁。碰撞死亡时屏幕出现回溯按钮，点击后角色回退至碰撞前 2 秒，黑白倒放动画。每局限 1 次，被列车追上不可用 | —（解锁型，无冷却） |
| S02 | 必杀技·虚空冲刺 | E | 角色向前冲刺 30m，路径上所有障碍物被摧毁，冲刺期间无敌 | 45 秒 |
| S03 | 幻影分身 | R | 在三条轨道上各生成一个半透明分身，与角色同步奔跑 6 秒，自动拾取各自轨道上的金币。金币全部归玩家 | 25 秒 |

> **技能设计原则**: S01 为分数解锁型死亡救急能力（到达 1500 分解锁），S02~S03 为常驻主动技能，覆盖「清障」「辅助」两个维度。回溯需在死亡瞬间手动点击触发，考验玩家反应。

### 2.5 增强体验（P1）

| 编号 | 功能 | 描述 |
|------|------|------|
| F15 | 金币收集 | 地面 + 浮空金币，幻影分身可跨轨道拾取 |
| F16 | 道具系统 | 仅保留双倍分数道具（拾取后 10 秒内分数翻倍），详见 §2.6 |
| F17 | 道具时长显示 | HUD 进度条显示双倍分数剩余时间 |
| F18 | 背景滚动 | 彩色几何体多层视差滚动 |
| F19 | 视觉反馈 | 纯几何体动画：死亡碎裂、塌陷闪烁预警、回溯黑白倒放、列车逼近红光 |
### 2.6 道具系统（P1）

> 道具系统保持简洁，仅保留**双倍分数**一种道具，聚焦技能系统作为核心差异化。

#### 2.6.1 道具定义

| 编号 | 道具名称 | 几何体造型 | 效果 | 持续时间 |
|------|----------|------------|------|----------|
| I01 | 双倍分数 | 黄色 Cube 旋转，顶面显示 "x2" | 获得分数翻倍 | 10 秒 |

#### 2.6.2 生成与拾取规则

| 规则 | 说明 |
|------|------|
| 生成方式 | 赛道上随机出现，平均每 400~600m 刷出 1 个 |
| 拾取方式 | 地面道具碰撞即拾取；浮空道具需跳跃碰撞拾取 |
| 视觉反馈 | 拾取后角色身边环绕黄色旋转 Cube，HUD 显示剩余时间进度条 |
| 时长显示 | 道具生效时 HUD 顶部显示 "x2" 图标 + 倒计时进度条，<= 3 秒时进度条变红闪烁 |
### 2.7 障碍物体系详表（P0）

障碍物分为三大类，随难度提升逐步解锁新类型。

#### 2.7.1 静态障碍（全程出现）

| 编号 | 类型 | 几何体造型 | 占轨道数 | 应对方式 | 颜色 |
|------|------|------------|----------|----------|------|
| O01 | 石块 | 1×1×1 Cube，随机偏移 | 1 | 变道绕开 | 深灰 |
| O02 | 墙体 | 3×3×0.5 拉伸 Cube，横跨轨道 | 2~3 | 跳跃跨越 | 暗红 |
| O03 | 尖刺 | 倒置小 Cylinder 阵列（3 个一排） | 1 | 跳跃跨越 | 红色 |
| O04 | 断台 | 轨道缺口（移除该段地面碰撞体） | 1~2 | 跳跃跨越 | — |

#### 2.7.2 动态障碍（500m 后解锁）

| 编号 | 类型 | 几何体造型 | 行为 | 应对方式 | 颜色 |
|------|------|------------|------|----------|------|
| O05 | 旋转机关 | 长条 Cube 绕中心匀速旋转 | 风车式旋转，覆盖 2 条轨道 | 找准时机跳跃/滑铲通过 | 暗黄 |

#### 2.7.3 陷阱机制（1000m 后解锁）

| 编号 | 类型 | 几何体造型 | 行为 | 后果 | 颜色 |
|------|------|------------|------|------|------|
| O08 | 掉落深坑 | 轨道大段缺口 + 下方无碰撞体 | 固定位置，覆盖 2~3 条轨道 | 掉落即死 | 黑色深渊 |
| O09 | 即死区域 | 红色 Cube 阵列（脉冲缩放动画） | 覆盖 1 条轨道，间歇激活 | 触碰即死 | 红色脉冲 |

> **生成约束**: 陷阱不可与静态障碍重叠。动态障碍速度随全局速度同步递增。始终保证至少 1 条轨道完全可通行。
### 2.8 数据持久化（P1）

| 编号 | 功能 | 描述 |
|------|------|------|
| F20 | 最高分记录 | 使用 PlayerPrefs 存储历史最高分 |
| F21 | 金币累计 | 总金币数持久化存储 |

## 3. 非功能需求

### 3.1 性能要求
| 指标 | 要求 |
|------|------|
| 帧率 | 稳定 ≥ 30 FPS（目标 60 FPS） |
| 内存占用 | ≤ 512 MB |
| 启动时间 | ≤ 5 秒 |
| Draw Call | ≤ 100（移动端如扩展需关注） |

### 3.2 可玩性要求
- 操作响应延迟 < 100ms（按下即反馈）
- 障碍物可见距离足够，给予玩家 ≥ 1 秒反应时间
- 难度曲线平滑递增，避免难度突变
- 障碍物类型通过颜色（红/橙）和形状（竖/横）明显区分

### 3.3 兼容性
- 支持 Windows 7 及以上系统
- 支持 16:9、16:10 宽屏分辨率自适应

### 3.4 可维护性
- 代码遵循 Unity 4.x C# 规范（无 async/await、无 ?.、无 string interpolation）
- 关键参数（速度、障碍间距、难度曲线）通过 public 变量暴露，便于调试
- 场景与 Prefab 结构清晰分层

## 4. Unity 4.6.8 技术约束

> 以下为使用 Unity 4.6.8 必须遵守的技术限制，直接影响需求实现方案。

### 4.1 C# 语言限制
| 不可用特性 | 替代方案 |
|------------|----------|
| `async/await` | 使用协程 `StartCoroutine()` |
| `?.` 空条件运算符 | 显式 `if (obj != null)` |
| `string interpolation ($"")` | `string.Format()` 或 `+` 拼接 |
| `dynamic` 类型 | 不使用 |
| `var` 在部分场景 | 显式声明类型 |

### 4.2 API 差异
| 现代 Unity API | Unity 4.x 等效 API |
|----------------|---------------------|
| `SceneManager.LoadScene()` | `Application.LoadLevel()` |
| `Cursor.lockState` | `Screen.lockCursor` / `Screen.showCursor` |
| `Self-Illuminated` 系列 Shader | `Self-Illumin/Diffuse` |
| 发射颜色 `_Emission` | `SetColor("_Illum", ...)` |
| `GetComponent<T>()` 泛型 | 可用（4.x 支持） |

### 4.3 物理注意事项
- **CharacterController**：地面碰撞体必须使用有厚度的 Cube，不能使用 Plane（会导致 `isGrounded` 不可靠）
- 避免在角色子对象上放置碰撞体（会推动 CharacterController）
- 角色初始 Y 坐标 ≥ 1.5，避免出生时穿透地面几何体
- Unity 4.x 无 `cc.minMoveDistance` 属性

### 4.4 资源导入
- 所有 `.cs` 文件使用 **CRLF 换行 + UTF-8 without BOM** 编码
- 序列化模式设为 `Force Text`，版本控制模式设为 `Visible Meta Files`
### 4.5 材质与美术约束

> **课程要求：不使用任何材质，不允许内置资源。** 所有渲染通过代码控制，不依赖任何 Material 资产文件。

| 约束 | 说明 |
|------|------|
| 禁止所有材质资源 | 项目中不得存在任何 `.mat` 文件，不得使用 Standard Assets、Asset Store、网络下载的任何 Material |
| 禁止内置默认材质 | 不得在 Inspector 中手动指定 Default-Material 等内置材质。GameObject.CreatePrimitive 自动附带的材质可接受 |
| 禁止贴图 | 不使用任何 `.png`/`.jpg` 等纹理文件 |
| 几何体着色 | 通过代码动态修改 `renderer.material.color` 实现颜色区分。所有颜色在脚本中以 `Color` 类型定义 |
| 道具与技能辨识 | 靠几何体形状 + 颜色 + Transform 动画（位移/旋转/缩放）区分，不依赖贴图 |
| 文字显示 | 使用 Unity 4.6.8 内置 uGUI Text 组件（Canvas + Text），不依赖位图字体 |

│   │   ├── TrapController.cs      — 陷阱激活与间歇逻辑
│   │   ├── CoinSpawner.cs         — 金币与浮空金币生成

│   ├── Skills/
│   │   ├── SkillManager.cs        — 技能冷却管理、全局技能状态

├── Scenes/

### 5.3 核心类职责

**PlayerController**
- 管理三轨道位置（X 轴：-2, 0, 2）、跳跃、滑铲、变道
- 处理死亡动画、回溯倒放动画
- 基于 CharacterController 物理交互

**GameManager**
- 状态机：Menu → Playing → Paused → Dead → [已解锁?] → Rewind(点击) → Playing / GameOver
- 自适应难度调控（综合列车距离 + 玩家表现）
- 分数累加（距离 × 速度系数 × 技能连击加成）
- 最高分与金币读写（PlayerPrefs）

**TrainController**
- 计算列车与玩家距离（初始 100m），距离 = f(玩家死亡次数、速度)
- 列车逼近时画面边缘红光渐强，距离 < 10m 触发即死

**AdaptiveDifficulty**
- 跟踪玩家生存表现（最近 5 次跑步的存活距离）
- 平均存活距离 > 2000m → 难度 +1；平均存活距离 < 500m → 难度 -1

**TerrainCollapse**
- 随机选择前方轨道方块，0.5s 黄色闪烁预警后下沉消失
- 3~5 秒后恢复升起，确保至少 1 条轨道始终稳固

**SkillManager**
- 管理技能的独立冷却计时、全局互斥
- 通知 HUD 更新技能冷却图标

**TimeRewind**
- 持续记录最近 3 秒的位置快照（每 0.1s 一帧）
- 碰撞死亡时屏幕弹出回溯按钮，点击后角色黑白倒放回退至碰撞前 2 秒
- 每局限 1 次，被列车追上不可用

**PhantomClone**
- 在三条轨道上各生成一个半透明角色副本，同步奔跑 6 秒
- 每个分身独立碰撞拾取各自轨道上的地面和浮空金币
- 分身无视障碍物碰撞，6 秒后消失

## 8. 自适应难度系统

> 摒弃固定阶段式难度，采用**实时自适应调控**。难度由列车追击压力 + 玩家近期表现共同决定。

### 8.1 难度参数

| 参数 | 范围 | 受什么影响 |
|------|------|------------|
| 全局速度 | 1.0x ~ 2.5x | 基础递进 + 难度等级加成 |
| 障碍密度 | 稀疏 ~ 密集 | 难度等级，高难度下多类型组合出现 |
| 塌陷频率 | 每 15s ~ 每 5s | 难度等级 |
| 列车追击距离 | 200m ~ 10m | 玩家死亡次数、速度 |
| 道具生成率 | 高 ~ 低 | 难度越高道具越少 |

### 8.2 自适应规则

| 条件 | 响应 |
|------|------|
| 最近 5 次跑步平均存活 > 2000m | 难度等级 +1（上限 10） |
| 最近 5 次跑步平均存活 < 500m | 难度等级 -1（下限 1） |
| 触发死亡回溯后 10 秒内 | 难度暂停变化，给予恢复期 |
| 列车距离 < 30m（危险） | 难度不再提升，维持当前 |
| 列车距离 > 150m（安全） | 难度加速提升至追赶 |

### 8.3 难度等级解锁

| 难度等级 | 障碍类型解锁 |
|----------|--------------|
| 1~3 | 静态障碍 O01~O04 |
| 4~6 | + 旋转机关 O05 |
| 7~8 | + 陷阱 O08~O09、塌陷频率提升 |
| 9~10 | 全部类型 + 组合出现 |
## 9. 美术风格

### 9.1 美术方向

> **核心约束**: 不使用任何材质（含内置默认材质）和贴图，所有视觉元素由 Unity 内置几何体组合而成。不使用粒子系统，不使用音频。颜色通过代码 renderer.material.color 控制。

| 元素 | 实现方式 | 颜色/动画 |
|------|----------|-----------|

## 11. 风险与应对

| 风险 | 可能性 | 应对措施 |
|------|--------|----------|
| CharacterController 地面检测不稳定 | 中 | 使用有厚度 Cube 做地面，不使用 Plane |
| 纯几何体无材质导致视觉单调 | 中 | 颜色+缩放动画+组合造型+技能特效丰富层次，所有颜色代码定义确保一致性 |
| 7 种障碍物 + 塌陷组合复杂度高 | 中 | 对象池统一管理，自适应难度控制密度上限 |
| 时间回溯与碰撞即死的节奏平衡 | 中 | 碰撞即死使游戏节奏紧凑，回溯解锁点（1500 分）需充分测试确保可达性 |
| 技能冷却与列车追击的平衡性 | 中 | 充分测试冷却时长和技能强度，参数外露便于调参 |
| 自适应难度可能导致难度抖动 | 中 | 设置平滑过渡和变化冷却期，避免边界跳变 |
